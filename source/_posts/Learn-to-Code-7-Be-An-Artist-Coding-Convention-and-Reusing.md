Coding convention

新成員加入、或程式碼的擁有人換人時，交接會更容易。

## 技巧五: Reusing
Reusing
減少重複的程式碼，讓程式碼更精簡、更容易閱讀、更容易被理解
可以讓一份程式碼被很多人或很多模組使用
修改程式碼或做擴充的時候會更有效率，只要改一兩個地方就好，不用到專案裡面的各個地方去改類似的功能
間接可以用有意義的名字把一段程式碼封裝起來
